# Refactor ModernUI JavaScript codebase into TypeScript

## Background
`extensions-builtin/sdnext-modernui` is currently written in JavaScript under `/javascript` and files are loaded individually by SD.Next.

## Scope
1. Work only inside extension repo path `extensions-builtin/sdnext-modernui`.
2. Do not modify SD.Next core repository files.
3. Loader integration and loader switch-over are out of scope for this task and handled manually.

## Non-goals
1. No changes to SD.Next core loader behavior.
2. No changes to runtime script registration/injection behavior in SD.Next.
3. No edits outside `extensions-builtin/sdnext-modernui`.

## Requirements
1. Create new `/src` folder and copy all ModernUI JavaScript source code into it, excluding vendor `src/vendor/split.js`.
2. Refactor all code in `/src` into TypeScript.
3. Keep `src/vendor/split.js` as vendor JavaScript.
4. Ensure vendor `split.js` is imported as needed and bundled inside final output `/javascript/modernui.mjs`.
5. Use `@vladmandic/build` (`^0.10.3`, which wraps `esbuild`) as the build runner to compile and bundle TypeScript into an ESM output file `/javascript/modernui.mjs` with source map `/javascript/modernui.mjs.map`.
6. Configure TypeScript as an exact copy of `extensions-builtin/sdnext-kanvas/tsconfig.json`, with a small documented delta allowed only for ModernUI-specific paths, names, or path aliases.
7. Configure build environment as an exact replica of `extensions-builtin/sdnext-kanvas` (toolchain, scripts, package-level build setup, `.build.json` profiles, and dependency/devDependency structure), using the same exact dependency and devDependency versions; unrelated package metadata fields (`name`, `author`, `repository`, `license`, `bugs`, `homepage`) must remain ModernUI-native, adapted only where ModernUI-specific paths or names require changes.
8. Omit the `konva` runtime dependency entirely; ModernUI has no equivalent and `dependencies` in `package.json` should be empty or absent.
9. Build profiles must match Kanvas exactly: `production` runs `compile` + `lint`; `development` runs `serve` + `watch` + `compile` + `lint`. The `serve` step must be adapted for ModernUI: use plain HTTP only (no SSL), point document root at the extension folder, no default file; document this as a required delta in `TS.md`.
10. Omit the `changelog` step from `.build.json`; do not include a `changelog` section.
11. Set `.build.json` `watch.locations` to `["src"]` (same as Kanvas, no change needed).
12. Set `.build.json` `build.global.banner` to a ModernUI-specific comment (e.g. `/*\n  SD.Next ModernUI — generated by @vladmandic/build\n*/\n`) so the output file is clearly identified.
13. Use `pnpm` as the package manager; commit `pnpm-lock.yaml` with exact Kanvas-pinned dependency versions as the install baseline.
14. Add lint and typecheck workflow for the extension (`eslint` + `tsc --noEmit`); `eslint` scope is `src/**/*.ts` only, excluding existing `/javascript` folder.
15. Add separate documentation file `TS.md` with:
	- migration summary
	- build commands
	- lint and typecheck commands
	- bundle output details
	- allowed `window` compatibility hooks whitelist
	- before/after global symbol diff snapshot summary
	- vendor `split.js` version/hash used for bundling
	- deterministic build expectation notes
	- tsconfig and `.build.json` delta list — enumerate confirmed adaptations (output path `dist/kanvas.esm.js` → `javascript/modernui.mjs`; serve config HTTP-only, no default file) and confirmed non-deltas (build target `platform`, `external` list, all TypeScript strictness settings identical to Kanvas)
	- config authority note: `tsconfig.json` is authoritative for `tsc --noEmit` typecheck; `.build.json` typescript section drives `@vladmandic/build` compile step
	- final exported global hooks list with assignment location
	- dependency drift guard note (policy for syncing toolchain versions when Kanvas updates)
	- split.js integration note: vendor file unchanged on disk, imported through source pipeline, bundled with preserved license banner
	- `pnpm-lock.yaml` policy: must be regenerated via `pnpm install` after any dependency version change; never manually edited

## Entry point
1. Canonical source entry path is `src/sdnext-modernui.ts` (migrated from `javascript/sdnext-modernui.js`).
2. Output is ESM: `/javascript/modernui.mjs`.

## Migration strategy
1. Migrate file-by-file from JavaScript to TypeScript modules.
2. After each migrated unit, verify behavior parity before proceeding.
3. Prefer TypeScript imports/exports and module-local scope over global variables where possible.
4. Do not introduce new runtime behavior toggles/flags during migration unless explicitly documented in `TS.md`.
5. Modules containing external fetch calls (`contributors.ts`, `server-info.ts`) must remain functionally identical; only add TypeScript type annotations, no behavioral changes.

## Constraints
1. Do not modify existing files in `/javascript` except creating new bundled output files `/javascript/modernui.mjs` and `/javascript/modernui.mjs.map`.
2. Existing JavaScript sources in `/javascript` must remain unchanged during migration.
3. New TypeScript implementation must preserve current behavior and be compatible for manual loader integration.
4. Build output must be ESM format; include JavaScript and external source map only; do not generate `.d.ts` files.
5. Do not increase global scope pollution; reduce existing global exposure where possible by using TypeScript module imports/exports and module-local scope.
6. Source map must be an external file referenced by the output bundle via `sourceMappingURL`; source map must use relative paths only (no machine-specific absolute paths); `sourcesContent` must be included.
7. Preserve vendor `split.js` license/banner attribution when bundled.
8. Build determinism is a documented expectation in `TS.md` (documentation check only, not a hard validation gate).

## Validation
1. `eslint` passes for new TypeScript and related new migration/build files.
2. `tsc --noEmit` passes.
3. Build succeeds and generates `/javascript/modernui.mjs` and `/javascript/modernui.mjs.map`.
4. Runtime behavior is preserved with no newly introduced global symbols on `window` beyond required compatibility hooks.
5. Global surface audit is documented: before/after `window` symbol diff and explicit whitelist compliance (documentation check only).
6. Minimal runtime smoke checks pass:
	- UI loads without console errors
	- split panes resize and persist
	- accordion state restore works
	- generate/enqueue/pause observer behavior works
	- server info refresh and copy works

## Deliverables
1. `/src` TypeScript source tree with `src/sdnext-modernui.ts` as entry.
2. TypeScript config exact to Kanvas config (with documented delta list).
3. Build environment and package-level setup exact to Kanvas with same exact dependency versions; ModernUI-native metadata fields retained.
4. `.build.json` with Kanvas-equivalent production and development profiles.
5. Lint/typecheck scripts.
6. `/javascript/modernui.mjs` and `/javascript/modernui.mjs.map` bundled ESM output.
7. `TS.md` documentation.
